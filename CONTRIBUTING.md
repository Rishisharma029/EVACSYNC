# Contributing to EvalSync

First off, thank you for considering contributing to EvalSync! It's people like you that make EvalSync such a robust and secure tool for handling high-traffic examination platforms.

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please create an issue on GitHub. Before creating an issue, please check if the issue has already been reported.

Include as much information as possible:
* Operating system and browser version.
* Steps to reproduce the bug.
* Any relevant logs or screenshots (check the audit logs and terminal logs in the UI).

### Suggesting Enhancements

We are always looking for ways to improve EvalSync. If you have an idea for a new feature or an enhancement, please create an issue on GitHub describing your idea.

### Pull Requests

1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (GitHub Actions will run CodeQL and Linters).
5. Issue that pull request!

## Code Style

* Use standard ES6+ JavaScript.
* Use CSS variables (defined in `:root`) for colors and spacing.
* Keep the code clean and well-commented, especially complex queuing logic.

## Security

If you discover a security vulnerability, please follow the guidelines in [SECURITY.md](SECURITY.md) instead of opening a public issue.
